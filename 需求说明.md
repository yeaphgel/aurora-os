一、你的核心需求（本质一句话）
===============

你要做的是：

> **构建一个可运行在 Hermes 或 OpenClaw 上的 Brand Content Runtime OS，实现企业品牌内容的每日自动生成、质检与发布闭环，并稳定调用 Image-2 生成视觉内容。**

* * *

二、系统目标（你真正想解决的问题）
=================

1. 内容生产自动化

----------

* 企业每天自动生成品牌内容
* 覆盖文案 + 图片 + 发布排期
* 无需人工设计与重复运营

* * *

2. 品牌一致性保证

----------

必须解决：

* Logo 必须稳定出现且位置固定
* 产品图必须正确嵌入
* 品牌色 / 字体 / 风格不能漂移
* 输出必须符合品牌规范

* * *

3. 爆款内容生成能力

-----------

系统需要：

* 基于产品 + 行业 + 活动生成内容选题
* 具有“爆款概率倾向”的内容决策机制
* 支持多内容类型分发（教育 / 情绪 / 转化 / 品牌）

* * *

4. 多 Agent 协作系统

---------------

需要拆分为：

* planner（选题）
* copywriter（文案）
* image2_adapter（图像生成）
* qa_checker（质量控制）

并由 Hermes / OpenClaw 进行调度

* * *

5. Image-2 能力集成

---------------

要求：

* 能基于文案自动生成视觉内容
* 支持品牌约束（风格/色彩/结构）
* 支持参考图（产品图/Logo）
* 输出可直接用于发布的成品图

* * *

6. 强制视觉嵌入系统（关键）

---------------

必须有一个独立机制：

* Logo 固定位置（不能由模型决定）
* 产品图强制嵌入
* UI/海报结构固定模板化
* 防止模型“自由发挥破坏品牌资产”

* * *

7. QA 质量控制系统

------------

必须包含：

* OCR 校验（文字是否正确）
* Logo 是否存在检测
* 产品图是否正确嵌入
* 品牌色一致性检测
* 输出合规性判断（fail / pass / retry）

* * *

8. 内容日历 + 发布系统

--------------

系统需要：

* 支持按天生成内容
* 基于营销活动（节日 / 促销 / 新品）
* 自动排期发布
* 支持多平台发布（小红书 / 微信 / 抖音 / LinkedIn）

* * *

9. Hermes / OpenClaw 集成需求

-------------------------

你希望：

* 只选择 Hermes 或 OpenClaw 作为宿主系统
* Brand OS 以插件或扩展形式运行
* 可以“像原生能力一样调用”
* 支持 runtime 方式插入 skill / workflow

* * *

三、系统能力拆解（功能模块）
==============

1. Brand Context（品牌资产层）

-----------------------

包含：

* Logo
* 产品图
* 品牌色
* 字体
* 语气风格
* 禁用词
* 内容规则

* * *

2. Content Strategy Engine（内容策略层）

---------------------------------

负责：

* 内容类型分配比例（教育/转化/情绪/品牌）
* 选题生成
* 爆款概率判断
* 内容方向决策

* * *

3. Skill Execution Layer（执行层）

-----------------------------

包含：

* planner.skill
* copywriter.skill
* image2_adapter.skill
* qa_checker.skill

* * *

4. Image-2 Adapter（图像生成接口）

--------------------------

负责：

* prompt结构化
* 调用 Image-2
* 绑定品牌约束
* 输出基础图

* * *

5. Deterministic Overlay System（确定性视觉层）

---------------------------------------

负责：

* Logo 强制嵌入
* 产品图强制嵌入
* 布局模板控制
* 不允许 AI 修改关键资产

* * *

6. QA Checker（质量系统）

-------------------

负责：

* OCR 文本一致性检测
* 品牌规则校验
* 图像完整性检测
* 自动 fail/retry

* * *

7. Publisher（发布层）

-----------------

负责：

* 多平台发布
* 定时任务
* 内容分发

* * *

四、系统运行流程（核心链路）
==============

    Brand Context Load        ↓Content Strategy Planning        ↓Copy Generation (planner + copywriter)        ↓Image-2 Generation (image2_adapter)        ↓Deterministic Overlay (logo + product)        ↓QA Checker (OCR + rules)        ↓Pass → Schedule PublishFail → Retry Loop

* * *

五、对 Hermes / OpenClaw 的要求
=========================

你希望系统具备：

1. 插件化能力

--------

* Brand OS 可“一键安装”
* 不修改核心系统
* 运行时加载

* * *

2. Runtime Hook 支持

------------------

支持：

* before_generate
* after_generate
* before_publish
* after_publish

* * *

3. 状态管理能力

---------

必须支持：

* brand session
* campaign session
* daily content session

* * *

4. Skill 可插拔

------------

* skill 可替换
* skill 可升级
* skill 可独立运行

* * *

六、核心设计原则（非常重要）
==============

1. AI只负责“决策”，不负责“最终资产控制”

------------------------

* Logo / 产品 = deterministic system
* AI不能直接控制最终视觉关键资产

* * *

2. 内容生产必须是“流程系统”，不是“生成工具”

-------------------------

* 多步骤 pipeline
* 可回溯
* 可 QA

* * *

3. Brand OS = 规则系统，不是生成系统

-------------------------

* 内容策略是核心资产
* 模型只是执行力

* * *

4. 模块必须可替换

----------

* Image-2 可升级
* skill 可替换
* 不破坏系统结构

* * *

七、你真正要做的产品（最终定义）
================

一句话版本：

> **一个运行在 Hermes / OpenClaw 上的 Brand Content Runtime OS，用于企业自动生成、校验并发布符合品牌规范的每日内容系统。**

* * *

八、如果再压缩成产品定位
============

你这个系统可以对外这样说：

> **企业品牌内容自动驾驶系统（Brand Content Autopilot OS）**

* * *

九、关键总结（帮你收敛认知）
==============

你现在做的不是：

* AI内容生成器
* Agent系统
* 插件工具

而是：

> **企业品牌内容生产的“操作系统层”**
